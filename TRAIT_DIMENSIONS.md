# Trait Dimension Mapping
**Complete reference for multi-dimensional scoring system**

## Magic Dimensions (6 total)

### healing-magic
**Description:** Preference for healing, protection, and support magic
**Related to:** Clerics, Life Domain, Paladins, support-focused characters
**Opposed by:** damage-magic (moderate), arcane-averse-magic (strong)
**Scoring examples:**
- "Want to heal allies?" YES → +4
- "Want damage spells?" NO → +2
- "Protective focus?" YES → +2

---

### damage-magic
**Description:** Preference for offensive and destructive spells
**Related to:** Wizards (Evocation), Sorcerers, Warlocks, combat-focused casters
**Opposed by:** healing-magic (moderate), arcane-averse-magic (strong)
**Scoring examples:**
- "Want damage spells?" YES → +4
- "Want to heal allies?" NO → +2
- "Elemental forces?" YES → +2

---

### utility-magic
**Description:** Preference for problem-solving, exploration, and practical magic
**Related to:** Wizards (all schools), Bards, versatile casters
**Opposed by:** arcane-averse-magic (moderate)
**Scoring examples:**
- "Magic for tricks?" YES → +4
- "Interested in magic?" MAYBE → +2
- "Scholarly pursuits?" YES → +2

---

### control-magic
**Description:** Preference for battlefield control, debuffs, and tactical magic
**Related to:** Wizards (Enchantment/Illusion), Bards, tactical casters
**Opposed by:** arcane-averse-magic (moderate)
**Scoring examples:**
- "Planning/strategy?" YES → +1 (tangential)
- "Want to heal allies?" NO → +1 (alternative support)

---

### versatile-magic
**Description:** Preference for broad magical knowledge across multiple schools
**Related to:** Wizards, Sorcerers, multi-school casters
**Opposed by:** arcane-averse-magic (strong)
**Scoring examples:**
- "Interested in magic?" YES → +4
- "Combine abilities?" YES → +2
- "Scholarly pursuits?" YES → +2

---

### arcane-averse-magic
**Description:** Preference for non-magical, physical approaches
**Related to:** Fighters, Barbarians, Rogues (non-Arcane Trickster), martial classes
**Opposed by:** All other magic dimensions (moderate to strong)
**Scoring examples:**
- "Interested in magic?" NO → +4
- "Physical excellence?" YES → +2
- "Weapon mastery?" YES → +1

## Scoring Value Reference

| Value | Meaning | When to Use |
|-------|---------|-------------|
| +4 | Strong alignment | Question directly asks for this preference |
| +3 | High alignment | Very related concept, strong connection |
| +2 | Moderate alignment | Related concept, clear connection |
| +1 | Slight alignment | Tangential or minor connection |
| 0 | Neutral | No impact or "don't know" |
| -1 | Slight rejection | Minor counter-indication |
| -2 | Moderate rejection | Related opposition |
| -3 | High rejection | Strong opposition |
| -4 | Strong rejection | Directly contradicts preference |

## Implementation Notes

1. **Balance Check:** For each question, sum all positive and negative potential scores. They should be roughly balanced to avoid trait inflation.

2. **Don't Know Handling:** `dont-know: {}` scores as 0 across all dimensions - truly neutral, doesn't affect min/max ranges.

3. **Specific Traits:** Can optionally remain binary (included in answer or not) OR be converted to scoring. Recommend keeping them as weighted presence for simplicity.

4. **Score Ranges:** Min/max for each trait calculated dynamically based on questions shown. Example:
   - If 5 questions can affect healing-magic with values: +4/-2, +2/-2, +1/0, +2/-1, +3/-2
   - Max possible: +4+2+1+2+3 = +12
   - Min possible: -2-2+0-1-2 = -7
   - Range: -7 to +12 (span of 19)

5. **Percentage Calculation:**
   ```
   If player scores +6 on healing-magic:
   percentage = (6 - (-7)) / (12 - (-7)) × 100
   percentage = 13 / 19 × 100 = 68%
   ```
